# plans/

This directory contains working documents: product requirements, architecture proposals, implementation summaries, and artifacts from AI-assisted development sessions.

These are living documents that reflect thinking-in-progress — not authoritative documentation. They capture decisions as they were being made, including paths not taken. Treat them as a project journal, not a specification.

For authoritative documentation, see:

- A2UI adapter system: [/lib/a2ui/README.md](../lib/a2ui/README.md)
